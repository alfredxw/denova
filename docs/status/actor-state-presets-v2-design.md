# Denova 内置状态系统预设 V2.3：分组字段与题材适配

> 状态：已实施（2026-07-19）
>
> 覆盖预设：默认、修仙、西幻、末世、无限流
>
> 影响范围：内置 Actor State 预设、状态结构树、剧情状态面板布局；不改变 Actor 状态更新、事件回放和故事级 Schema 冻结机制

## 1. 结论

状态系统只维护“本回合结束后仍成立，并会影响后续行动”的事实。

本版采用以下设计：

1. “面板”和“状态”是字段分组，不再是两个需要编辑纯 JSON 的 object 字段。
2. 每一项都是既有 Schema 支持的普通字段；结构树使用“模板 → 分组 → 字段”的简单嵌套。
3. 剧情状态栏直接按相同分组渲染普通字段，不需要理解面板 object 内部结构。
4. 面板保存相对稳定、会参与检定的属性；状态保存资源、持续效果和冷却等动态信息。
5. 五套预设只共享模板边界，不共享一套固定数值。每个题材独立决定是否存在面板，以及面板和状态包含什么。
6. 数值字段必须同时提供上下界和区间语义，不能只给一个没有解释的数字。
7. 技能、物品、关系、任务、地点和势力仍使用 object 记录集合，因为它们的条目数量和稳定 ID 是动态的；这与把固定面板字段藏进 JSON 是不同场景。
8. 字段 `group` 只提供初始分组，最终展示顺序由用户在当前故事的状态面板中调整，不写死在 Schema。

预设不保存纪要、总结、完整经历、行动选项、幕后计划和回合流水。这些继续由 Turn、Director、Lore、choices 等组件负责。

## 2. 分组普通字段模型

### 2.1 Schema 形态

默认 TRPG 的面板与状态使用普通字段：

```json
{
  "fields": [
    {
      "name": "等级",
      "type": "number",
      "default": 1,
      "min": 1,
      "max": 20,
      "group": "面板",
      "display": "stat"
    },
    {
      "name": "生命",
      "type": "string",
      "default": "10/10",
      "group": "状态",
      "display": "inline"
    },
    {
      "name": "持续效果",
      "type": "list",
      "default": [],
      "group": "状态",
      "display": "list"
    }
  ]
}
```

这意味着：

- 配置界面分别编辑“等级”“生命”“持续效果”，不编辑一个巨大的面板或状态 JSON。
- 规则绑定可直接引用 number 字段，不依赖内嵌数值路径。
- 状态栏沿用现有 number、string、list 渲染器。
- 用户自定义字段只需选择分组，不需要学习新的子字段协议。

### 2.2 结构树

状态结构树按字段首次出现的分组顺序展示：

```text
主角状态表
├─ 人物设定
│  ├─ 姓名
│  └─ 基本身份
├─ 面板
│  ├─ 等级
│  ├─ 力量
│  └─ 攻击 AC
├─ 状态
│  ├─ 生命
│  ├─ 持续效果
│  └─ 冷却状态
└─ 持有与关系
   ├─ 技能与能力
   ├─ 重要物品
   └─ 关系
```

没有填写 `group` 的自定义字段统一进入“未分组字段 / Ungrouped Fields”，不会从结构树消失。

### 2.3 面板与状态边界

面板字段满足两个条件：

- 相对稳定，通常由等级、长期能力、装备或持续状态修正；
- 会作为检定或规则结算的输入。

状态字段满足以下任一条件：

- 当前资源会消耗或恢复；
- Buff、Debuff、伤势、异常或规则影响会开始、叠加、衰减或解除；
- 技能或物品存在冷却、次数或恢复条件。

“当前处境”仍是叙事字段，描述优势、危险、限制和直接压力，不与生命、法力或 Buff 混在一起。

## 3. 固定模板边界

五套预设都使用五种模板：

| Template ID | 名称 | 实例方式 | 内容 |
| --- | --- | --- | --- |
| `story_context` | 故事状态表 | 单例 | 当前场景、可承接钩子、世界局势、当前任务、题材级状态 |
| `protagonist` | 主角状态表 | 单例 | 人物设定、题材面板、题材状态、技能、物品、关系 |
| `important_character` | 重要角色状态表 | 每角色一个 Actor | 角色资料、题材面板与状态、对主角关系及自身持有记录 |
| `opponent` | 敌对对象状态表 | 每对象一个 Actor | 威胁、题材面板与状态、能力、弱点及持有记录 |
| `world_entities` | 世界实体表 | 单例 | 地点和势力 |

默认只创建三个 Actor：

| Actor ID | Template ID |
| --- | --- |
| `protagonist` | `protagonist` |
| `story` | `story_context` |
| `world` | `world_entities` |

重要角色和敌对对象按需创建，不预建空 Actor。

## 4. 公共角色字段

角色模板共享的是信息归属，不是固定面板数值。

### 4.1 主角

| 分组 | 字段 |
| --- | --- |
| 人物设定 | 姓名、基本身份、外貌与装扮、性格与背景 |
| 面板 | 由题材预设决定 |
| 状态 | 由题材预设决定，另保留当前处境 |
| 持有与关系 | 技能与能力、重要物品、关系 |

### 4.2 重要角色

| 分组 | 字段 |
| --- | --- |
| 人物设定 | 基本身份、外貌与装扮、性格与背景 |
| 状态 | 出场与位置、当前目标与处境，以及题材状态 |
| 面板 | 由题材预设决定 |
| 主角关系 | 与主角关系、对主角好感度、对主角的已知信息 |
| 持有与关系 | 技能与能力、重要物品、对其他目标的关系 |

不建立“对主角的怀疑”或“误解”专用字段。已经产生行动影响的内容进入“与主角关系”“对主角的已知信息”或“当前目标与处境”。

### 4.3 敌对对象

| 分组 | 字段 |
| --- | --- |
| 威胁 | 类型与外观、存续与位置、威胁态势 |
| 面板 | 由题材预设决定 |
| 状态 | 行动模式、弱点与退场条件，以及题材状态 |
| 持有与关系 | 技能与能力、重要物品、关系 |

“威胁等级”和“威胁状态”合并为“威胁态势”，使用低、中、高、致命四档并附带当前行为描述。

## 5. 精简后的故事状态

故事状态公共部分由 9 个字段缩减为 7 个：

| 分组 | 字段 | 类型 | 用途 |
| --- | --- | --- | --- |
| 当前场景 | 当前时间 | string | 日期、时段或阶段，只保留有行动意义的精度 |
| 当前场景 | 当前详细地点 | string | 具体地点及必要的上级范围 |
| 当前场景 | 当前事件 | string | 正在发生的事件、直接压力和眼前问题 |
| 当前场景 | 在场角色 | list | 当前可感知或正在互动的 Actor ID |
| 当前场景 | 可承接钩子 | string | 下一段可以直接承接的一个未完成动作、对话、发现或迫近威胁 |
| 世界状态 | 世界局势 | string | 跨场景威胁、环境变化和倒计时 |
| 当前任务 | 当前任务 | object | 已成立且仍需推进的任务记录 |

删除：

- 当前场景要素：真正重要的对象进入当前事件、可承接钩子、物品、地点或任务。
- 当前生效规则：长期规则属于 Lore 或题材设定，临时规则影响进入当前事件或题材状态。
- 世界背景：稳定世界设定由 Lore 维护，状态表只保留会变化的世界局势。

“可承接钩子”只保留一个最直接的承接点，不写剧情总结，不生成选择列表，也不使用“继续探索”之类的空泛表达。钩子兑现、失效或被更强的新钩子替代时更新。

## 6. 题材适配

### 6.1 面板与状态字段

| 预设 | 面板分组 | 状态分组 | 明确不做 |
| --- | --- | --- | --- |
| 默认 TRPG | 等级、力量、敏捷、体质、智力、感知、魅力、攻击 AC、防御 DC | 生命、法力、持续效果、冷却状态 | 不把基础值、当前值、修正说明再嵌套进 object |
| 修仙 | 当前境界 | 境界进度、持续状态、冷却状态 | 不默认量化身法、体魄、根骨、悟性、神识、气运；灵根和特殊体质使用文本 |
| 西幻 | 职业、等级、攻击 AC、防御 DC | 生命、施法资源、持续效果、冷却状态 | 不默认继承六维；作品采用其他属性时由用户增加 |
| 末世 | 默认没有面板分组 | 生存状态、持续效果、冷却状态 | 不制造空面板；不把饥饿、口渴、疲劳、感染、污染拆成一排低价值字段 |
| 无限流 | 空间评级 | 当前资源、规则影响、冷却状态 | 不自行生成六维或战力评分；没有公开评级时保持“未公布” |

题材字段同时应用于主角、重要角色和敌对对象。题材不需要的字段不会以空 object 占位。

### 6.2 题材附加字段

| 预设 | 角色附加 | 故事附加 |
| --- | --- | --- |
| 修仙 | 灵根与特殊体质、根基与传承、本相与传承特征 | 修行世界状态 |
| 西幻 | 超凡来源与血脉契约、阵营职责、生物特性与抗性 | 魔法、信仰与政治秩序 |
| 末世 | 生存专长与职责、感染与变异特征 | 灾变与生存局势、基地状态 |
| 无限流 | 空间身份、队伍角色、规则实体特征 | 轮回空间状态、当前副本 |

### 6.3 字段规模

| 预设 | 主角 | 故事 | 重要角色 | 敌对对象 | 世界实体 |
| --- | ---: | ---: | ---: | ---: | ---: |
| 默认 | 21 | 7 | 24 | 21 | 2 |
| 修仙 | 13 | 8 | 16 | 13 | 2 |
| 西幻 | 17 | 8 | 20 | 17 | 2 |
| 末世 | 11 | 9 | 15 | 12 | 2 |
| 无限流 | 13 | 9 | 16 | 13 | 2 |

字段数增加只发生在原 object 被展开为有独立含义、可以直接展示和绑定的普通字段时；不以“顶层字段越少越好”为目标，也不创建没有独立更新时机的重复字段。

## 7. 数值量表

所有内置 number 字段都提供 `min`、`max` 和区间解释。

### 7.1 好感度

| 区间 | 语义 |
| ---: | --- |
| 0–20 | 敌视或强烈排斥 |
| 21–40 | 戒备或疏远 |
| 41–60 | 中立、熟悉或可合作 |
| 61–80 | 友好且愿意信任 |
| 81–90 | 亲近并会主动支持 |
| 91–100 | 深厚羁绊；除非发生根本冲突通常不会动摇 |

一般事件变化 1–5，重大选择变化 6–15；普通同场、寒暄或没有关系结果的互动不自动增加。好感度不等同于信任、服从或恋爱关系。

### 7.2 默认 TRPG

| 字段 | 区间语义 |
| --- | --- |
| 等级 | 1–4 初阶；5–10 中阶；11–16 高阶；17–20 传奇 |
| 六维 | 1–5 极弱；6–9 偏弱；10–11 常人；12–15 优秀；16–19 卓越；20–30 超凡 |
| 攻击 AC / 防御 DC | 0–5 极低；6–9 偏低；10–13 常规；14–17 高；18–30 极高 |

### 7.3 修仙境界进度

| 区间 | 语义 |
| ---: | --- |
| 0–20 | 初入 |
| 21–50 | 积累 |
| 51–80 | 深化 |
| 81–99 | 临界 |
| 100 | 具备突破条件，不代表必然突破成功 |

## 8. 动态记录约定

技能、物品、关系、任务、地点和势力继续使用 object，map key 直接使用故事语言中稳定、可读的名称作为 ID。状态系统只校验根值为 object，不要求子项必须是 object，也不补写或校验重复的内部名称字段；下列列名只是给 Agent 的内容引导。关系记录的 key 仍使用目标 Actor 或势力的精确 ID。

| 记录 | 主要列 | 默认上限 |
| --- | --- | ---: |
| 技能与能力 | 类型、掌握或当前状态、效果、代价与限制、来源 | 主角 12；其他 6 |
| 重要物品 | 类型、数量或状态、作用、限制、来源 | 主角 16；重要角色 6；敌对对象 4 |
| 关系 | 类型、阶段、好感度、态度、边界或承诺、主要矛盾 | 主角 12；重要角色 6；敌对对象 4 |
| 当前任务 | 类型、状态、目标与进度、时限、奖惩、关联 ID | 8 |
| 地点记录 | 类型、所属范围、状态、风险、特征、通路 | 16 |
| 势力记录 | 类型、特征、存续、立场、范围、行动 | 8 |

物品由存放它的 Actor 持有；转移时同轮从原持有者移除并写入新持有者。关系表示当前 Actor 对目标的单向视角，不自动镜像。

## 9. 状态面板布局与 DnD

### 9.1 Schema 与布局边界

- `fields` 数组顺序只是首次展示的兜底顺序。
- `group`、`display` 是默认展示提示。
- 新预设不生成 `order` 或 `display_groups`。
- 用户布局按“故事 ID + Template ID”保存在浏览器本地，不进入 Story、Actor State、提示词或模型上下文。
- 同一故事中使用同一模板的 Actor 共享布局；不同故事互不影响。

### 9.2 拖拽行为

状态面板的布局编辑器支持：

1. 拖动整个分区调整分区顺序。
2. 拖动字段调整组内顺序，或移动到其他已有分区。
3. 拖动过程即时预览，但只在松手后持久化一次，避免连续写本地偏好造成抖动。
4. 分区拖动只与分区碰撞；字段拖动先锁定指针所在分区，再在该分区内选择字段目标，避免父子可放置区域竞争。
5. 使用独立拖动浮层，源项目保持占位；空分区提供明确放置区域。
6. 保留键盘拖拽，以及上移、下移、前一分区、后一分区按钮。
7. 一键恢复 Schema 兜底布局。

Schema 新增字段时，字段追加到其默认分区；删除字段时自动忽略旧布局记录。

## 10. 兼容与实施边界

1. 保留五个内置预设 ID、五个模板 ID，以及 `story`、`protagonist`、`world` 三个基础 Actor ID。
2. 保留“当前详细地点”和“当前事件”字段，继续满足运行时基础依赖。
3. 已开始故事继续使用开局时冻结的故事级 Schema，不自动迁移为本版分组字段。
4. 新结构只影响新故事及用户主动恢复的内置预设；同 ID 的用户覆盖不自动替换。
5. 本版会移除内置预设中的 `面板` / `状态` object 字段，改为题材对应的普通字段；这是 Beta Schema 的有意不兼容变更。
6. `order`、`display_groups` 的忽略只影响展示，不改变状态值或回放结果。
7. 布局偏好保存在浏览器本地；清理站点数据或更换设备后会回到默认布局。
