# 互动模式
在我们小说ide的左上角，支持模式切换开关，可以切换到互动模式。
互动模式特点：
- 左侧是角色设定(character.md)+世界观设定(world-building.md)+提示词设定（CREATOR.md）三个模块可切换展示。
  - 支持通过agent对话来初始化或修改角色设定和世界观设定，和当前情况类似
- 中间是故事舞台，支持agent流式输出
- 右侧是当前实时状态，类似 progress.md，但不太一样的是需要同时维护当前故事状态，角色状态。而character.md和world-building.md是静态文件，不需要维护，除非用户主动要求修改。
- 故事舞台下方是输入框让用户可以输入，agent会根据用户输入来生成故事，也可以维护当前故事状态，角色状态。
  - 支持切换agent模式
    - 默认是互动模式即用户和故事进行交互的模式，这个模式用户可以和故事互动，每一轮对话都会落盘，落到一个文件中不分章节，上下文可以持续维护，agent会根据用户输入来生成故事，也会维护当前故事状态，角色状态 在 story-{id}.jsonl 中以事件流的形式存在
    - 可以切换到人设和世界观设定修改模式，用户可以在这个模式下和agent探讨修改角色设定和世界观设定，agent会根据用户输入来修改character.md和world-building.md
  - 其他能力同当前创作agent对话相似，支持输出中断，skills，风格引用等功能
- 最底部是剧情时间线 / 分支树，回到任意回合分叉, 且分支并行可见

架构设计：
- agent 不能用和 ide 的 agent 同样的提示词，代码基本可以复用，但是需要做好隔离，避免和ide的agent冲突
- ide模式下的风格引用/版本管理/设置/等能力保留，用户可以在互动模式下继续使用这些功能

# 左侧设计
左侧角色 可以拖到中间 "召唤上场", 离开舞台时角色 "退场"—— 影响 prompt 的注入

# 数据设计
同一本书支持来回切换两个模式，agent交互的目录不同
- 共享的东西是 setting 里面的世界观和角色设定，但是故事线不需要关联
- IDE模式是 setting/outline.md，setting/progress.md
- 互动模式下 故事存储于 interactive/story/story-{id}.jsonl，人设和世界观设定修改模式下是 setting/characters.md，setting/world-building.md
  - interactive/story/story-{id}.jsonl，每个id代表不同的开端的故事线，类似不同对话，数据是 单个 jsonl 追写，数据结构上支持分支切换，通过id和parentid来实现分支切换，类似 git 分支切换，dag形式
  - 状态嵌入到统一的 story-{id}.jsonl 中，存的是事件流delta（角色状态和关键事件），右侧看到是当前分支状态快照
- setting/characters.md，setting/world-building.md 这两个文件是不同模式共享的


用户目录 .nova/ 下增加 interactive 目录，里面结构：
- config.toml 用户配置
- data/ 存储用户数据的目录
- story-tellers 预设好可以选择的不同的故事讲述者，对应不同风格的故事，用户可以根据需要选择不同的讲述者或自己创建
  - 每个讲述者都有自己的提示词设定，用户可以根据需要选择不同的讲述者
  - 每个讲述者都有自己的风格，自己会给story加入不同的随机事件，让用户体验不同的故事线路
